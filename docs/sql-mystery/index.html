SQL Mystery Getting Started

If you're confident in your SQL, you can skip these explanations and put your skills to the test!

What's in the database?

An ERD, which stands for Entity Relationship Diagram, is a visual representation of the tables in a database. Here's the ERD for the mystery database:

If two tables are related, the matching columns, i.e. the ones you use in a JOIN, are connected by a gray arrow in the diagram.

If you were to look at the data in this database, you would see that the tables are huge! There are so many data points; it simply isnâ€™t possible to go through the tables row by row to find the information you need.

For example, how many people are in the database?

SELECT count(*)
FROM person;

What do we know about these people?

SELECT * FROM person LIMIT 10;

What are the possible values for a column?

SELECT DISTINCT type
FROM crime_scene_report;

Go For It!

You have enough info to start solving the mystery. You'll need to check the ERD and make some reasonable assumptions, but there's nothing else you need!
